# .perso

Suivi personnel du projet — séparé du code pour ne pas polluer la racine du repo.

- `TODO.md` — idées futures / backlog
- `ROADMAP.md` — historique des phases livrées
- `DEPLOY_RUNNER.md` — mise en place du runner self-hosted (CI/CD vers le Raspberry Pi)
- `audits/` — rapports d'audit (sécurité, dépendances, etc.), un fichier par audit, nommé `YYYY-MM-DD-sujet.md`
